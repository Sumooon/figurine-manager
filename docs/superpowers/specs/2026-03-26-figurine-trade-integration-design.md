# 手办编辑弹窗集成交易信息

## 背景

当前编辑手办和编辑交易信息是分离的操作，用户需要在不同页面间切换才能完成相关操作，体验繁琐。

## 需求

在编辑手办时，能够同时查看和编辑该手办的交易信息，无需跳转页面。

## 设计方案

### 布局变更

- 弹窗宽度：600px → 800px
- 右侧表单区域底部新增可折叠的"交易信息"面板
- 使用 `el-collapse` 组件，默认折叠状态

### 数据模型

- 一个手办最多一条活跃交易记录（isActive=true）
- 交易信息与手办通过 `figurineId` 关联

### 交互逻辑

1. **展开面板时**：
   - 自动查询该手办的交易记录
   - 有记录则回填，无记录则显示空表单

2. **保存逻辑**（串行执行，确保数据一致性）：
   - 先保存手办信息
   - 若交易面板已展开且卖出价 > 0：
     - 有交易记录 → 更新
     - 无交易记录 → 新增
   - 交易保存成功后，自动将手办状态设为"已出"

3. **删除交易记录**：
   - 交易面板展开时显示"删除交易"按钮
   - 删除前需二次确认
   - 删除成功后：
     - 移除该手办的交易记录
     - 自动将手办状态从"已出"改为"在售"

4. **面板标题**：
   - 无交易记录：`▶ 交易信息`
   - 有交易记录：`▶ 交易信息 - 已售 ¥XXX`

### 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 用户手动选择"在售"但填写了交易信息 | 以交易信息为准，状态强制为"已出" |
| 手办保存成功但交易保存失败 | 提示错误，不自动改状态，保持手办已保存的状态 |
| 新增手办时交易面板 | 默认折叠，无特殊提示 |
| 卖出价为0或未填写 | 不创建/不更新交易记录 |

### UI 结构

```
左侧 (33%)       右侧 (67%)
┌──────────┐    ┌────────────────────────┐
│  图片预览 │    │ 基本信息表单           │
│  图片选择 │    │ ...                    │
└──────────┘    │────────────────────────│
                │ ▶ 交易信息（折叠面板）  │
                │   [展开后显示交易表单]  │
                │   [删除交易] 按钮       │
                └────────────────────────┘
```

### 交易信息字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 卖出价 | number | 必填，> 0 时才保存交易 |
| 总成本 | display | 从手办信息读取，只读 |
| 咸鱼手续费 | number | 自动计算 |
| 实际收入 | number | 自动计算 |
| 利润 | number | 自动计算 |
| 利润率 | number | 自动计算 |
| 卖出时间 | datetime | 默认当前时间 |
| 备注 | text | 可选 |

### 文件变更

1. `FigurineForm.vue` - 主要修改
   - 增加交易信息折叠面板
   - 集成交易数据的加载和保存逻辑
   - 增加删除交易按钮和逻辑
   - 弹窗宽度调整为 800px
   - 添加利润样式（`.profit-positive`、`.profit-negative`）

2. `stores/trade.ts` - 新增方法
   - `getActiveTradeByFigurineId(figurineId)` - 获取指定手办的活跃交易记录（包装 db 方法）
   - `removeTrade(tradeId)` - 删除交易记录

3. `TradeForm.vue` - 保持不变
   - 仍用于交易记录页面的独立编辑入口
   - 利润计算逻辑已通过 `calculateTradeFinancials` 共享

### 技术细节

- 复用 `calculateTradeFinancials` 工具函数计算利润
- 复用 `db/trade.ts` 的 `getActiveTradeByFigurine` 方法
- 保存时串行执行：先手办后交易，确保数据一致性
- 删除交易时同步更新手办状态
